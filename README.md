Agent两个框架大致流程（Agent流程）

一个**简单**智能体SimpleAgent**（含工具）的**完整生命周期**：首先用户发送消息给到Agent，Agent负责构建上下文消息，先找Memory记忆层获取历史消息来拼接成完整上下文消息，再发送给LLM（大脑）来处理，处理完后来**解析**判断要不要调用工具（条件判断），调用后返回消息给LLM整理上下文后再发送给Agent（LLM二次调用只负责整理最后的回答，转换成自然语言），如此循环，最后由Agent判断不用再调用工具/给LLM后返回结果给用户，并且将上下文信息记录给Memory（History）

- 结果是一次直接输出，不用循环反复的思考和更新提示词让LLM重新思考（LLM只会调用一次）



**ReActAgent** 的 Think-Act-Observe 循环：Agent 首先分析当前的任务和已有的信息发给LLM，基于上一步的思考，LLM 决定调用哪一个具体的**工具（Action）**，Agent 运行工具，并获取工具返回的真实数据（Observation），Agent 将这一轮的“思考+行动+观察”全部存入历史记录。在下一轮循环开始时，换个词再搜一次（再发送给LLM进行进一步优化处理）

- 结果是多次循环调用LLM后反复优化Prompt和思考结果后才返回的



Agent只是一个执行者，是一个抽象概念，它包含了大模型（LLM）+ 规划（Planning）+ 记忆（Memory）+ 工具使用（Tool Use）

五个基类形成一个**完整的依赖链**：Config（配置） 和 Message（上下文信息） 是最底层的**数据**类，LLM 依赖它们，Agent 组合 LLM + Config + Memory(Message[]) + ToolRegistry(Tool[]) 构成完整的**运行单元**。
