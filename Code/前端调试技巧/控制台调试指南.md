## 三、Console 控制台 - 第一优先级

### 1. 常见错误类型与等级

```javascript
// 🔴 红色错误 (Error) - 程序中断，必须修复
Uncaught TypeError: Cannot read properties of undefined
Uncaught ReferenceError: xxx is not defined
SyntaxError: Unexpected token

// ⚠️ 黄色警告 (Warning) - 不会中断但要注意
Deprecated feature
Warning: xxx

// 🔵 蓝色信息 (Info) - 正常日志
console.log() 输出的内容
```

### 2. 错误信息解读公式

```
错误名称: 错误描述
    at 函数名 (文件名:行号:列号)
    at 函数名 (文件名:行号:列号)
    at ...
```

**实战例子：**
```
Uncaught TypeError: Cannot read properties of undefined (reading 'data')
    at ChatApp.vue:25:15
    at Proxy.mounted (ChatApp.vue:20:8)
```

**解读步骤：**
1. **错误类型**：`TypeError` - 类型错误
2. **具体问题**：读取 `undefined` 的 `data` 属性
3. **位置**：`ChatApp.vue` 第 25 行，第 15 列
4. **调用栈**：从 `mounted` 钩子函数触发的

### 3. 点击文件名快速跳转

在控制台中点击蓝色的文件链接（如 `ChatApp.vue:25:15`），会直接跳转到出错位置！

## 四、Network 网络 - 第二优先级

### 1. 查看 API 请求状态

| 状态码 | 颜色 | 含义 | 常见原因 |
|--------|------|------|---------|
| 200 | ✅ 绿色 | 成功 | 正常 |
| 400 | ⚠️ 黄色 | 客户端错误 | 参数错误 |
| 401 | ⚠️ 黄色 | 未授权 | Token 失效 |
| 403 | ⚠️ 黄色 | 禁止访问 | 权限不足 |
| 404 | ⚠️ 黄色 | 未找到 | 接口地址错误 |
| 500 | 🔴 红色 | 服务器错误 | 后端报错 |
| (failed) | 🔴 红色 | 请求失败 | 跨域、网络问题 |

### 2. 查看 API 请求详情

点击某个请求后，可以看到：

#### Headers（请求头）
```
Request URL: http://localhost:8000/api/chat
Request Method: POST
Status Code: 200 OK
```

#### Payload（请求体）
```json
{
  "messages": [{"role": "user", "content": "你好"}],
  "user_id": 1,
  "conversation_id": 123
}
```

#### Preview/Response（响应内容）
```json
{
  "code": 0,
  "message": "success",
  "data": {...}
}
```

### 3. 常见网络问题排查

| 问题 | 排查方法 |
|------|---------|
| CORS 跨域错误 | 查看 Console 中 `Access-Control-Allow-Origin` 错误 |
| 请求发送但无响应 | 检查后端服务是否运行，端口是否正确 |
| 参数错误 | 查看 Payload 中的请求体是否正确 |
| 响应格式错误 | 查看 Response 是否符合预期格式 |

## 五、Elements 元素 - 第三优先级

### 1. 检查 DOM 是否正确渲染

```html
<!-- 预期结构 -->
<div id="app">
  <div class="chat-container">...</div>
</div>

<!-- 如果看不到，说明渲染失败 -->
```

右侧 Styles 面板可以：
- 查看元素应用了哪些样式
- 检查样式是否被覆盖（划掉的样式）
- 临时修改样式测试效果

### 3. Vue/React DevTools

安装浏览器扩展后，可以查看组件状态：
- **Vue DevTools**：查看 Vue 组件的 data、props、computed
- **React DevTools**：查看 React 组件的 state、props

## 七、常见错误速查表

### 前端错误

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `Cannot read properties of undefined` | 访问未定义对象的属性 | 检查对象是否为空，加可选链 `?.` |
| `xxx is not defined` | 变量/函数未声明 | 检查拼写，import 是否正确 |
| `Unexpected token` | 语法错误 | 检查括号、引号是否匹配 |
| `Failed to fetch` | 网络请求失败 | 检查跨域、后端服务 |
| `Maximum call stack size exceeded` | 无限递归 | 检查递归终止条件 |

### API 错误

| 状态码 | 含义 | 常见原因 |
|--------|------|---------|
| 400 | 请求参数错误 | 参数类型/格式不对 |
| 401 | 未授权 | Token 缺失或过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | URL 路径错误 |
| 500 | 服务器内部错误 | 后端代码异常 |
| 502 | 网关错误 | 后端服务未启动 |

## 八、实用调试技巧

### 1. 使用 console.log 调试

```javascript
// 打印变量值
console.log('用户数据:', userData)

// 打印对象（格式化）
console.table(usersArray)

// 分组打印
console.group('用户信息')
console.log('姓名:', name)
console.log('年龄:', age)
console.groupEnd()

// 条件断点
console.assert(age >= 18, '年龄必须大于18')
```

### 2. 使用 debugger 断点

```javascript
function processUser(user) {
  debugger; // 程序会在这里暂停
  return user.name;
}
```

### 3. 清空控制台

| 操作 | 方法 |
|------|------|
| 清空控制台 | 点击 🚫 图标 |
| 清空 Network | 右键 → Clear browser cache |
| 过滤请求 | 上方过滤框输入关键词 |

### 4. 过滤错误信息

控制台上方可以过滤：
- `All` - 全部
- `Errors` - 只看错误
- `Warnings` - 只看警告
- `Info` - 只看信息
- 自定义过滤 - 输入关键词

## 九、当前项目常见问题

### 后端 `http://localhost:8000`

1. **确认后端运行**
   ```bash
   # 浏览器访问
   http://localhost:8000/health
   # 应该返回 {"status": "ok"}
   ```

2. **查看后端日志**
   - 运行 `python run.py` 的终端输出
   - 查看报错堆栈

3. **前端静态文件**
   - 确认 `static/dist/` 目录存在
   - 确认 `index.html` 文件存在

### 前端 API 请求

检查前端配置的 API 地址：
```javascript
// 应该指向
baseURL: 'http://localhost:8000/api'
```

## 十、快速检查清单

遇到问题时按顺序检查：

- [ ] 打开 F12 开发者工具
- [ ] 查看 Console 有无红色错误
- [ ] 查看 Network 请求状态码
- [ ] 确认后端服务正在运行
- [ ] 确认前端静态文件已构建
- [ ] 查看后端终端日志

- [ ] 清除浏览器缓存
